## ✅ 任务 1：统一备份导入阈值（已完成）

### 问题描述
- **位置**: `packages/worker/src/services/settings-service.ts`
- **问题**: 
  1. 使用字符长度而不是字节大小（不准确）
  2. 所有类型使用统一阈值（120,000 字符）
  3. 图片片段可能超过文本限制导致导入失败
  4. 导航链接总数限制过严（6,000）
- **优先级**: P1（高优先级）

### 修复方案
采用**按类型分类 + 字节大小**策略：
1. 统一使用字节大小而不是字符长度
2. 按内容类型设置不同的限制
3. 放宽限制以满足用户需求（用户体验优先）
4. 提供友好的错误提示（显示当前大小和限制）

### 实施细节

**新的限制配置**:
```typescript
const CONTENT_SIZE_LIMITS = {
  source: 10 * 1024 * 1024,  // 订阅源：10MB（兼容大型订阅源）
  text: 5 * 1024 * 1024,      // 纯文本：5MB（支持长文档）
  code: 1 * 1024 * 1024,      // 代码片段：1MB（支持完整代码文件）
  image: 10 * 1024 * 1024,    // 图片：10MB（支持高清截图）
  link: 10 * 1024              // 链接：10KB（足够存储元数据）
} as const;

const MAX_BACKUP_BYTES = 100 * 1024 * 1024;  // 总备份：100MB
const MAX_BACKUP_ITEMS_PER_SECTION = 2_000;   // 每分类条目数
const MAX_BACKUP_TOTAL_NAV_LINKS = 10_000;    // 导航链接总数（从 6000 提升）
```

**改进点**:
1. ✅ 统一使用字节大小（`TextEncoder().encode().byteLength`）
2. ✅ 按类型分类限制（source/text/code/image/link）
3. ✅ 放宽限制（图片 10MB，文本 5MB，订阅源 10MB）
4. ✅ 导航链接总数从 6,000 提升到 10,000
5. ✅ 友好的错误提示（`formatBytes` 函数）
6. ✅ 显示当前大小和限制值

**代码变更**:
```typescript
// 修改前：
const MAX_BACKUP_TEXT_LENGTH = 120_000; // 字符长度
if (source.content.length > MAX_BACKUP_TEXT_LENGTH) {
  throw new Error(`订阅源内容超长，单条最大 ${MAX_BACKUP_TEXT_LENGTH} 字符`);
}

// 修改后：
const contentBytes = new TextEncoder().encode(source.content).byteLength;
if (contentBytes > CONTENT_SIZE_LIMITS.source) {
  throw new Error(
    `订阅源内容过大，单条最大 ${formatBytes(CONTENT_SIZE_LIMITS.source)}` +
    `（当前 ${formatBytes(contentBytes)}）`
  );
}
```
