## 技术亮点

### 1. 依赖注入模式
- 清晰的依赖关系
- 易于测试（可以 mock 依赖）
- 避免循环依赖

### 2. 分布式锁机制
- 防止并发刷新
- 锁超时保护（2 分钟）
- 锁竞争日志

### 3. 缓存一致性检查
- 防止节点缓存和格式化缓存不一致
- 自动恢复机制

### 4. 递归展开订阅
- 支持嵌套订阅（最多 2 层）
- 防止循环引用
- 并发控制（最多 8 个并发）

### 5. SSRF 防护
- 协议白名单（http/https）
- Localhost 黑名单
- 内网域名黑名单
- IPv4/IPv6 保留地址黑名单
- DNS Rebinding 攻击防护

### 6. 双后端支持
- D1 优先（生产环境）
- KV 降级（开发环境）
- 统一的接口抽象

### 7. 按类型分类的内容限制
- 订阅源：10MB
- 图片：10MB
- 笔记：5MB
- 代码片段：1MB
- 链接：10KB

### 遇到的挑战

1. **URL 规范化陷阱**
   - `file:///etc/passwd` 被规范化为 `https://file:///etc/passwd`
   - 解决：在规范化前先检查协议

2. **测试数据结构**
   - 需要理解 API 的数据结构
   - 解决：仔细阅读代码和类型定义

3. **复杂度评估**
   - 阶段 6 的复杂度超出预期
   - 解决：及时调整计划，暂缓高风险任务

4. **依赖注入实现**
   - 需要修改多个函数签名
   - 解决：使用包装函数传递依赖
